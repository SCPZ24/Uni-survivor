# auto-report

自动完成中式大学实验报告。核心目标是：**先尊重模板，再补齐内容；像学生写报告，而不是像论文或技术白皮书。**

## 工作流

1. **盘点材料。** 先查看工作区里是否有实验指导书、实验数据、截图、代码、参考结果、往届报告或空白模板。不要先编内容。
2. **检测报告模板。** 优先识别文件名含 `模板`、`实验报告`、`report`、`实验`、`课程报告`、`实践报告` 的 `.docx`、`.wps`、`.doc`、`.pdf`、`.md`、`.tex` 文件；也要留意目录名如 `template/`、`模板/`、`report/`。
3. **有模板时复制副本。** 先把模板复制成新的报告文件，再只修改副本。不要覆盖原模板。副本命名优先使用课程、实验名、姓名/学号等已有信息；缺信息时用 `<原文件名>-已完成`。
4. **无模板时新建报告。** 按常见中式大学实验报告结构写：实验名称、实验目的、实验原理、实验环境、实验内容/步骤、实验结果与分析、实验总结。用户或材料给出结构时，以材料为准。
5. **填充正文。** 从材料中提取真实信息；缺少数据时可以合理补写过程性描述，但不要伪造精确数值、截图结果、姓名学号、日期、教师姓名。需要占位时用清晰占位符。
6. **做格式自检。** 交付前检查模板保护、样式一致、学生口吻、内容完整性。

## 模板处理硬规则

- **发现模板就必须复制。** 所有编辑都发生在新副本上，原模板保持不变。
- **保留对应字体和字号。** 有模板时，报告里所有标题、正文、表格文字、图题、表题、页眉页脚等，都要沿用模板中对应位置或对应样式的字体、字号、加粗、缩进、行距和编号习惯。
- **不要重排模板。** 不要为了“更好看”改封面、目录、标题层级、页边距、页眉页脚、表格结构、段前段后或编号格式，除非用户明确要求。
- **找不到完全对应样式时就近匹配。** 例如新增二级标题时，复制模板中已有二级标题的样式；新增正文段落时，复制模板普通正文样式；新增表格内容时，沿用同表格相邻单元格样式。
- **PDF 模板只作版式参考。** 如果模板是 PDF 且不能可靠原位编辑，按 PDF 的栏目、顺序、标题层级、字体大小观感重建可编辑版本，并说明原 PDF 未被覆盖。
- **别动信息栏和评分栏** 如果看到模板里有个人信息页，成绩评分页或教师评语页，除非用户明确要求，否则不要改动这些内容。

## 写作口吻

- 用普通本科/专科学生的口吻写，允许使用“我”或“我们”。
- 表达要自然、朴素、具体：像认真做完实验后的报告，不像高水平论文、厂商技术文档或 AI 总结。
- 结论和总结可以写真实学习感受：例如“通过本次实验，我们熟悉了……”“在调试过程中我发现……”“后续还可以……”
- 不要过度专业化。少用宏大抽象词、管理咨询式套话、复杂长句和过密术语。
- 不要写成检讨、心得体会或宣传稿。实验报告仍要有步骤、结果、分析和总结。

## 内容补全规则

| 部分 | 写法 |
| --- | --- |
| 实验目的 | 用 2-4 条说明要掌握的知识、工具或方法，贴近课程要求。 |
| 实验原理 | 讲清与本实验直接相关的概念、公式、流程或代码逻辑，篇幅适中。 |
| 实验环境 | 写操作系统、软件、语言、依赖、设备型号等材料中能确认的信息。 |
| 实验步骤 | 按实际操作顺序写，使用“首先、然后、最后”等学生报告常见表达。 |
| 实验结果 | 优先放真实截图、输出、表格、图像或运行结果；没有结果材料时保留占位提示。 |
| 结果分析 | 解释现象是否符合预期，指出误差、问题、调试过程或原因。 |
| 实验总结 | 用“我/我们”总结收获、问题和改进方向，不要写得太满。 |

## 质量闸门

交付前逐项确认：

- 已搜索工作区模板；如果找到模板，已复制副本并只改副本。
- 模板副本中的标题、正文、表格、图题、表题字体和字号与模板对应样式一致。
- 原模板没有被覆盖或破坏。
- 报告结构符合模板或实验指导书要求。
- 正文使用学生口吻，至少在总结或分析中自然使用“我”或“我们”。
- 没有伪造不可确认的姓名、学号、精确数据、截图结果、教师评价或日期。
- 缺失信息已用明显占位符标出，方便用户后续补齐。
